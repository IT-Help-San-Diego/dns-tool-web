DEVONagent Pro Plugin

Add intelligence-grade DNS security analysis to your DEVONagent Pro research workflows. Each query returns up to 13 enriched results — security reports, reconnaissance data, integrity-verified snapshots, and provenance metadata.

When a domain is analyzed through this plugin, you'll see a Plugin tag next to the domain in your analysis history.

What You Get

  • 1. Engineer's DNS Intelligence Report — full security analysis with live scanning
  • 2. Covert Recon Report — full analysis in covert operations mode
  • 3. Observed Records Snapshot — reconstructed zone file, SHA-3-512 verified
  • 4. Covert Security Badge — scotopic-optimized embeddable badge
  • 5. Detailed Security Badge — full posture score badge
  • 6. Analysis Pipeline & Protocol Map — DNS topology visualization
  • 7. Zenodo DOI — permanent scientific record (Concept DOI: 10.5281/zenodo.18854899)
  • 8. Full JSON Intelligence Data — complete analysis payload download
  • 9. SHA-3 Integrity Checksum — cryptographic verification of analysis data
  • 10. Subdomain CSV Export — reconnaissance data in spreadsheet format
  • 11. Security Remediation Plan — actionable remediation steps
  • 12. Executive Report — board-ready executive summary
  • 13. Sources & Methodology — RFC citations, data sources, and scoring methodology

Download & Install

Download Plugin (.plist) Download Plugin (.json)
Why two formats? Every built-in DEVONagent plugin (Google, arXiv, OpenRadar, etc.) uses Apple's native plist (Property List) format — it's the documented, canonical format with full feature support including categories, crawl delays, and XML/Atom result parsing. DEVONagent also accepts JSON files in the Plugins directory, which is simpler to read and edit. Both work identically for our plugin. The plist is recommended if you want to match the native format.

Quick Install (drag and drop)

1
Download the plugin file

Click either download button above. The .plist format is recommended.

2
Move it to the Plugins folder

Place the downloaded file in:
~/Library/Application Support/DEVONagent/Plugins/

To open this folder: in Finder, press ⇧⌘G and paste the path above.

3
Restart DEVONagent Pro

Quit and relaunch DEVONagent Pro. The plugin will appear in Preferences → Plugins & Scanners (⌃⌘P).

4
Enable in your search set

Open your preferred search set and check the box next to DNS Tool - Domain Security Intelligence. It will appear under the Security category. Query any domain name to begin.

How It Works

When DEVONagent queries a domain through this plugin, it sends a GET request to the /agent/search endpoint. Because ParseLinks is enabled, DEVONagent extracts every <a href> from the returned HTML as a separate search result — giving you direct access to every report, badge, export, and data artifact from a single query.

The __agentQuery__ placeholder in the engine URL is automatically replaced by DEVONagent with whatever domain you search for.

Plugin Anatomy

DEVONagent plugins are defined by a small set of configuration keys. Here's what each one does:

KeyPurpose
NameDisplay name shown in the plugin list and search sets.
IdentifierUnique identifier for the plugin. Convention is reverse-DNS or the service domain.
CategoryFolder group in the Plugins & Scanners window (plist only). Ours uses Security.
EngineUrlThe search endpoint. __agentQuery__ is replaced with the search term. __agentOffset__ handles pagination.
ParseLinksWhen true, DEVONagent extracts every <a href> from the HTML response as a separate result.
ResultsPerPageExpected number of results per page. Set to match the number of links your endpoint returns.
KeywordArray of trigger words. DEVONagent may prioritize this plugin when queries match.
OperatorsBitmask for supported search operators (phrase, Boolean, etc.). 59 enables standard operators.
Start / OffsetPerPagePagination controls. For single-page results, set Start: 0 and OffsetPerPage: 1.

Plist vs JSON — Format Comparison

Plist (.plist)JSON (.json)
StatusNative format — used by all built-in pluginsSupported — DEVONagent reads JSON from the Plugins directory
Category supportYes — plugins appear under named groups (Security, Science, etc.)No — appears ungrouped
XML/Atom parsingFull support via ResultXPath, TitleXPath, etc.Not applicable to our HTML-based plugin
Human readabilityVerbose XML — harder to edit by handClean and compact — easy to edit
macOS toolingNative — editable with Xcode, plutil, defaultsStandard — any text editor
Community adoptionAll known community plugins use plistDNS Tool is among the first to ship JSON

For our ParseLinks-based plugin, both formats are functionally identical. The plist gives you the Category grouping (your plugin appears under "Security" in the sidebar). JSON is simpler to read and share. We provide both — use whichever you prefer.
